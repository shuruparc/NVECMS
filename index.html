The Mission of NVE is to build a website that allows for teachers to easily manage their lessons, so students are able to receive the maximum contact time available from their teachers. It is also to allow editors and publishers to dissimentate resources to teachers efficiently without any issues from a CMS. This will be ably supported by proper documentation to help users understand how to use the platform and how it benefits the end user.

The Vision of NVE is to make the system programming get as close to real time practical deployment as far as possible so that editors do not have any performance issues, and teachers are able to retrieve information they want. Eventually, NVE will become a dynamic website that will allow teachers to create lessons with multiple resources and present them how they wish to.

What NVE CMS does?

A)   Admin User
B)   Staff / Editor User
C)   Teacher User

The CMS has a resource pool which we call a Resource Hub. The providers of this pool of resources are all the 3 users mentioned above. The data is stored with Title, Stage (KS1, KS2 etc) and Subject filters.

This enables the CMS end user i.e the teachers to easily filter resources based on Stage and Subject and pull out information for their lesson preparation. Resources are the key to this CMS and elimination of redundancy becomes important for the end user. The resources which are taken from Online links are converted to Tiny URLs which we call as the Link resolver. Thereafter the CMS stores browser information and their environment which we call as a Systems Checker database which is stored for every user and accessed only by the admin.

The principal idea of NVE CMS is to integrate all the users into a System which is extremely user friendly, Scalable and Robust in nature.
